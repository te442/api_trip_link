# סעיף 21 – קוד התוכנית

**פרויקט:** Trip Link – מערכת תכנון טיולים  
**טכנולוגיה:** ASP.NET Core REST API (C#) + SQL Server  
**תאריך:** יוני 2026

---

## הקשר כללי

מערכת **Trip Link API** היא שרת REST שמשרת אפליקציית Angular. הליבה העסקית של המערכת היא **תכנון מסלול טיול אופטימלי** – בחירת יעדים, סידורם בסדר ביקור, וחישוב זמני נסיעה בתחבורה ציבורית.

האלגוריתם מורכב מ-Pipeline של 7 שלבים (Step 0–6), עם **Simulated Annealing (SA)** לשיפור המסלול, ונתוני תחבורה מ-**Google Maps Transit API**.

### תרשים זרימה כללי

```
Angular Client  ──HTTP/JSON──►  Controllers
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
              AuthService    TripService    OptimizerService
                    │               │               │
                    ▼               ▼               ▼
               SQL Server      SQL Server    OptimizerPipeline
                                              (Step 0 → Step 6)
                                                    │
                                                    ▼
                                          Google Maps Transit API
```

### תרשים Pipeline האופטימיזציה

| שלב | שם | תפקיד |
|-----|-----|-------|
| Step 0 | INPUT | טעינת טיול, סינון יעדים, בניית פרמטרים |
| Step 1 | WEIGHTS | חישוב משקלים לכל יעד |
| Step 2 | SCORE_TABLE | בניית טבלת ציונים תלת-ממדית (מקור→יעד→שעה) |
| Step 3 | RANK | דירוג יעדים לפי משקל |
| Step 4 | INITIAL_ROUTE | בניית מסלול ראשוני |
| Step 5 | SA_LOOP | Simulated Annealing – אופטימיזציה (עד 50,000 איטרציות) |
| Step 6 | OUTPUT | בניית תוכנית טיול, Narrative בעברית |

```csharp
public async Task SaveOptimizedRouteAsync(int tripId, List<int> destinationIds)
{
    // מוחק יעדים קיימים ומוסיף לפי סדר האופטימיזציה
    var existing = _context.DesOfTrips.Where(d => d.TripId == tripId);
    _context.DesOfTrips.RemoveRange(existing);

    for (int i = 0; i < destinationIds.Count; i++)
    {
        _context.DesOfTrips.Add(new DesOfTrip
        {
            TripId      = tripId,
            DesId       = destinationIds[i],
            VisitNumber = i + 1
        });
    }
    await _context.SaveChangesAsync();
}
```

**הסבר:** אחרי שהמשתמש מאשר המסלול האופטימלי, הפונקציה מעדכנת טבלת `DesOfTrips` – מוחקת יעדים קודמים ושומרת הסדר החדש עם `VisitNumber`.

```csharp
                + crowdScore   * 0.10   // עומס / תיירות

                + transitScore * 0.50;  // יעילות תחבורה (המשקל הגבוה!)
return Normalize(rawScore);
```

**הסבר:** נוסחת הציון משקללת ארבעה מרכיבים. יעילות תחבורה (transit) מקבלת 50% – כי המערכת מתמקדת בטיולים בתחבורה ציבורית.

#### פונקציה 5: `ExecuteAsync` – Step 5 (Simulated Annealing) – הליבה!

| | |
|---|---|
| **מיקום** | `Services/Optimizer/Steps/Step5_SaOptimizer.cs` |
| **קלט** | `OptimizationContext` (InitialRoute, ScoreTable, Destinations) |
| **פלט** | עדכון `ctx.BestRoute`, `ctx.SaResult` |
| **תפקיד** | אופטימיזציה של המסלול באמצעות Simulated Annealing |

```csharp
for (int i = 1; i <= MaxIterations && temperature > MinTemperature; i++)
{
    var neighbor = GetNeighborRoute(currentRoute, ctx, routeBuilder);
    if (!neighbor.IsValid)
    {
        rejected++;
        temperature *= (1.0 - CoolingRate);
        continue;
    }

    if (AcceptSolution(neighbor, currentRoute, temperature))
    {
        currentRoute = neighbor;
        accepted++;

        if (CombinedCost(neighbor) > CombinedCost(bestRoute))
            bestRoute = neighbor.Copy();
    }
    else { rejected++; }

    temperature *= (1.0 - CoolingRate);
}
ctx.BestRoute = bestRoute;
```

**פרמטרים:** `MaxIterations = 50,000`, `CoolingRate = 0.003`, `MinTemperature = 0.01`

**הסבר:** זה הלב של האלגוריתם. Simulated Annealing יוצר "מסלול שכן" (neighbor) – מוסיף/מסיר/מחליף יעד – ובודק אם לקבל אותו. פתרונות טובים מתקבלים; פתרונות גרועים עשויים להתקבל בהסתברות שקטלת עם הטמפרטורה. כך המערכת מוצאת מסלול אופטימלי תוך כדי יציאה מפתרונות מקומיים.

**פעולות שכן (Neighbor):**
- הוספת יעד חדש למסלול
- הסרת יעד מהמסלול
- החלפת מיקום של שני יעדים

```csharp
public async Task<AuthResponseDto> RegisterAsync(RegisterDto dto)
{
    if (string.IsNullOrWhiteSpace(dto.Email) || string.IsNullOrWhiteSpace(dto.Password))
        throw new ArgumentException("אימייל וסיסמה הם שדות חובה");

    var email = dto.Email.Trim().ToLowerInvariant();
    if (await _context.Users.AnyAsync(u => u.Email == email))
        throw new InvalidOperationException("כתובת האימייל כבר רשומה במערכת");

    var user = new User
    {
        UserId       = Guid.NewGuid().ToString(),
        FullName     = dto.FullName.Trim(),
        Email        = email,
        PasswordHash = BCrypt.Net.BCrypt.HashPassword(dto.Password)
    };

    _context.Users.Add(user);
    await _context.SaveChangesAsync();
    return BuildAuthResponse(user);
}
```

**הסבר:** בודקת שדות חובה, מוודאת שאימייל לא קיים, יוצרת משתמש עם סיסמה מוצפנת (BCrypt), שומרת ב-DB ומחזירה JWT Token.
